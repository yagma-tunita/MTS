                NET 网络服务模块说明

================================================================================

本模块位于 backend/net 目录，负责 HTTP 基础设施，包括中间件、安全防御和路由注册。
所有 HTTP 请求先经过 net 层，再转发至 internal/handler 处理业务逻辑。

--------------------------------------------------------------------------------
1. 目录结构
--------------------------------------------------------------------------------
net/
├── middleware/               # HTTP 中间件
│   ├── auth.go               # JWT 认证中间件（支持角色校验）
│   ├── cors.go               # 跨域中间件（可配置）
│   ├── logger.go             # 请求日志（自动生成请求ID）
│   ├── rate_limit.go         # IP 限流（令牌桶算法）
│   └── recovery.go           # Panic 恢复中间件
├── protect/                  # 安全防御组件
│   ├── security_headers.go   # 安全响应头（CSP, X-Frame-Options 等）
│   ├── honeypot.go           # 蜜罐（阻断恶意扫描路径）
│   ├── ip_blocklist.go       # IP 黑名单
│   └── request_guard.go      # 请求限制（body大小、URL长度、方法白名单）
└── router/
    └── router.go             # 路由注册（组装所有中间件和 handler）

--------------------------------------------------------------------------------
2. 中间件功能
--------------------------------------------------------------------------------
2.1 auth.go
    - 从 Authorization 头提取 Bearer token
    - 验证 JWT，解析出 user_id, username, role
    - 将用户信息存入 Gin 上下文
    - 提供 RequireRole() 方法进行角色授权

2.2 cors.go
    - 使用 gin-contrib/cors 库
    - 默认允许所有来源（开发环境），生产环境可配置白名单

2.3 logger.go
    - 自动生成或透传 X-Request-ID
    - 记录 method, path, status, latency, client_ip, user_agent
    - 使用 pkg/logger 输出结构化日志

2.4 rate_limit.go
    - 基于 IP 的令牌桶限流
    - 默认 100 令牌/秒，突发 20
    - 超出返回 429 Too Many Requests

2.5 recovery.go
    - 捕获 panic，记录堆栈日志
    - 返回 500 Internal Server Error

--------------------------------------------------------------------------------
3. 防御组件功能
--------------------------------------------------------------------------------
3.1 security_headers.go
    - 设置 X-Content-Type-Options, X-Frame-Options, X-XSS-Protection,
      Referrer-Policy, Content-Security-Policy

3.2 honeypot.go
    - 内置常见攻击路径（/wp-admin, /phpmyadmin, /admin 等）
    - 匹配路径返回 404 并记录日志，不影响正常业务路由
    - 支持配置开关、自定义路径、返回状态码

3.3 ip_blocklist.go
    - 从配置加载 IP 黑名单
    - 拒绝黑名单 IP 访问，返回 403

3.4 request_guard.go
    - 限制请求体最大字节数（默认 4MB）
    - 限制 URL 最大长度（默认 2048）
    - 限制 HTTP 方法白名单
    - 可选屏蔽特定 User-Agent（子串匹配）

--------------------------------------------------------------------------------
4. 路由注册（router.go）
--------------------------------------------------------------------------------
- 按照顺序加载全局中间件：
    Logger → Recovery → CORS → SecurityHeaders → Honeypot → IPBlocklist → RequestGuard → RateLimit
- 定义 /health 健康检查端点
- 分组 /api/v1
    - /auth 公开路由（登录、刷新令牌）
    - 其他路由受 JWT 保护，并根据角色授权
- 404 处理函数

依赖注入：router.Setup(h *handler.Handlers, jwtSvc jwt.JWTService) 接收业务 handler 和 JWT 服务。

--------------------------------------------------------------------------------
5. 使用方式
--------------------------------------------------------------------------------
在 main.go 中调用：
    r := router.Setup(handlers, jwtSvc)
    r.Run(":" + cfg.Server.Port)

所有中间件和防御组件均提供 DefaultXXXConfig() 工厂函数，可按需覆盖配置。

--------------------------------------------------------------------------------
6. 注意事项
--------------------------------------------------------------------------------
- 本模块不包含任何业务逻辑，仅提供基础设施。
- 所有中间件均无状态，可水平扩展。
- 生产环境应修改 CORS 允许的域名，并开启 IP 黑名单。
- 限流中间件基于内存，多实例部署时需改用 Redis 实现分布式限流。
- 蜜罐路径可根据实际攻击日志动态调整。
